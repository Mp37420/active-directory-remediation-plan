# Active Directory Security Remediation Plan

## 📌 Project Overview
Conducted a comprehensive security assessment of an Active Directory environment and developed a structured remediation plan to address critical vulnerabilities impacting security, compliance, and operational integrity.

This project demonstrates hands-on experience in identifying real-world enterprise security gaps and aligning remediation strategies with industry frameworks.

---

## 🎯 Objectives
- Identify security vulnerabilities within Active Directory infrastructure  
- Assess risks associated with misconfigurations and legacy systems  
- Develop a remediation roadmap aligned with best practices  
- Improve overall security posture and reduce attack surface  

---

## 🔍 Key Findings
- Outdated and unsupported operating systems  
- Group Policy Preference (GPP) password exposure  
- IPv6, LLMNR, and NBT-NS poisoning vulnerabilities  
- Kerberoasting and AS-REP Roasting risks  
- SMBv1 protocol usage  
- Sensitive data exposure and weak access controls  

---

## 🛠️ Remediation Approach
- Implemented system patching and OS upgrades  
- Enforced secure credential management (LAPS, gMSA)  
- Disabled insecure protocols (LLMNR, SMBv1)  
- Strengthened access control and privilege management  
- Recommended monitoring and logging improvements  

---

## 📊 Impact
- Reduced potential attack vectors across the network  
- Improved compliance with ISO 27001 and NIST frameworks  
- Strengthened identity and access management controls  
- Enhanced protection of sensitive organizational data
